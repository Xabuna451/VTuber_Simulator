# 스트리밍 vs 휴식 밸런스 가이드

목적

- 프로그래머/디자이너/테스터가 스트리밍(Streaming)과 휴식(Rest) 활동의 밸런스를 빠르게 이해하고 조정할 수 있도록 숫자·목표·테스트 케이스·튜닝 포인트를 정리한다.

요약 (한줄)

- 스트리밍: 스트레스↑, 피로↑, 행복 소폭↑(mood 영향), 건강 약간↓ — 위험 누적형
- 휴식: 스트레스 크게↓, 피로 크게↓, 행복↑, 건강↑ — 회복형

현재 구현 핵심 (DeltaStats.cs 기준)

- Tick = 1시간, 하루 처리 시간 = 8 Tick
- Streaming per-tick 변화(주요 식)
  - dStress = (4 + 6*f - 5*(h-0.5)) \* Random(0.3~0.6) [+2 if stress>70]
  - dFatigue = 5 + 5\*s
  - dHappiness = 8 \* (mood - 0.5) (추가로 stress에 따라 -2/-4)
  - dHealth = -(2*f + 1.5*s) (+ 보정: happiness>0.6 시 약간 회복)
- Rest per-tick 변화(주요 식)
  - dStress = -(3 + 4*s + 2*f) \* Random(0.7~1.2)
  - dFatigue = -(4 + 6*f) * Random(0.8~1.3)
  - dHappiness = (1 + 3*max(s,f)) * Random(0.8~1.2) (단, 행복·건강이 매우 높으면 절반)
  - dHealth = (2 + 3*f + 2*s) \* Random(0.8~1.2)

예상 실전 수치 예시 (초기 상태: S=20,F=20,H=80 -> s=f=0.2,h=0.8)

- Streaming (per tick)
  - dStress ≈ 1.1 ~ 2.2 → 하루(8tick): +8.8 ~ +17.6
  - dFatigue ≈ 6 → 하루: +48
  - dHappiness ≈ +1.6 → 하루: +12.8
  - dHealth ≈ -0.7~+0.05 → 하루: -5.6 ~ +0.4 (행복 보정에 따라)
- Rest (per tick)
  - dStress ≈ -2.9 ~ -5.0 → 하루: -23.5 ~ -40.3
  - dFatigue ≈ -4.2 ~ -6.8 → 하루: -33.6 ~ -54.4
  - dHappiness ≈ +1.28 ~ +1.92 → 하루: +10.2 ~ +15.4
  - dHealth ≈ +2.4 ~ +3.6 → 하루: +19.2 ~ +28.8

디자인 목표(권장)

- 스트레스: 플레이어가 연속으로 3일 연속 스트리밍하면 위험(예: 50~80) 구간 도달 가능 → 휴식 필요성 유도
- 피로: 하루 스트리밍 후 피로가 눈에 띄게 오르도록(예: +30~50/주) → 휴식 1~2일로 회복 가능
- 행복/건강: 스트리밍은 소폭 상승/감소, 휴식으로 회복이 충분히 이루어져야 함
- 플레이 선택성: 스트리밍 연속 선택은 보상(구독자 상승 등)을 주되, 리스크(이벤트/컨디션 악화) 유발

튜닝 포인트(어디를 조정할지)

- 스트레스 랜덤 스케일 (Streaming: 0.3~0.6)
  - 줄이면 스트레스 덜 누적 → 플레이어가 더 많이 스트리밍 가능
  - 늘리면 리스크 상승
- 피로 기본값(Streaming dFatigue = 5 + 5\*s)
  - 상수(5) 증가 → 피로 누적 빨라짐
  - s 계수(5) 변경 → 피로가 높을수록 더 빠른 악화/회복 조절
- 휴식 회복 계수(휴식 dFatigue / dStress / dHealth)
  - 회복량을 줄이면 플레이어가 더 많은 휴식 필요
  - 늘리면 게임 페이스가 느슨해짐
- 행복 보정(Streaming 및 Rest 내부의 행복 관련 보정)
  - 행복의 영향 범위를 좁히거나 확대하여 '기분'의 중요도 조절

플레이테스트 시 체크할 KPIs

- 평균 주당 스트리밍 일수 (목표: 3~4일로 유지)
- 플레이어가 강제 휴식을 하지 않아 컨디션 붕괴(예: stress>90 또는 health<20)로 이어지는 빈도 (목표: 낮음, 단 위험 발생은 존재해야 함)
- 회복 속도: 스트레스/피로가 보통(50 근처)에서 안전(20~30)으로 내려가는 데 걸리는 일수(목표: 휴식 1~2일)
- 선택형 이벤트 빈도와 선택 결과로 인한 상태 변동이 게임 리듬을 깨지 않는지

테스트 시나리오 (간단)

1. 연속 스트리밍 시나리오
   - 7일 전부 Streaming으로 채움 → 결과: stress, fatigue 가 급증 → 중간 이벤트(강제 휴식/성급한 실패) 발생 권장
   - 통과 기준: 플레이어는 7일 만에 치명적 상태로 잘못 도출되지 않음(대신 리스크 이벤트 발생)
2. 번갈아 스트리밍/휴식 시나리오
   - 1일 Streaming → 1일 Rest 반복 → 결과: 안정적 성장(행복 유지, 피로 관리)
   - 통과 기준: 안정적으로 피로/스트레스 유지, 건강 상승
3. 극단적 상황
   - 초기 stress=80 상태에서 3일 Streaming → 결과: stress>90 또는 강제 휴식/부작용 트리거 발생
   - 통과 기준: 게임이 명확한 경고/강제 대응을 제공

디버그/로깅 제안 (개발자에게 요청할 것)

- Streaming/Rest Tick 로그: dS,dF,dHap,dHe 와 누적 후 상태 (현재 이미 로그 존재)
- 이벤트 발생 시 로그 레이블에 day/week/slot 정보 포함
- Playtest 빌드에서는 랜덤 시드 고정 옵션 추가(재현성 테스트용)

UI 피드백 권장 (아티스트/UX에 전달할 메시지)

- 슬롯 클릭 시 예상 효과 툴팁: "Streaming(예상): +스트레스 +피로, +행복(조건)"
- 확정(Confirm) 후 요약 모달: 주간 예상 변화(스트레스/피로/건강 예상 합계) 표시
- 위험 레벨 시각화: 스트레스/건강 바 색상 변화(녹→노랑→빨강) 및 경고 아이콘

빠른 튜닝 레시피 (숫자 제안)

- 더 안전하게: Streaming stressScale 범위를 [0.2,0.5], dFatigue base 4 → 스트레스·피로 완화
- 더 도전적으로: Streaming stressScale [0.4,0.8], dFatigue base 6 → 위험 증가
- 휴식 완화: Rest fatigueScale [0.6,1.0] (더 빠른 회복)
- 휴식 강화: Rest fatigueScale [0.9,1.4] (더 강한 회복)

권장 우선 조정 순서 (테스트 효율성 기준)

1. 스트리밍 stressScale 범위 조정 (랜덤성 → 안정성 영향 큼)
2. Streaming dFatigue base 값 조정 (피로 누적 속도)
3. Rest dFatigue / dStress 스케일 조정 (회복력 조절)
4. Happiness 보정(Streaming/Rest) 조정 (행복이 건강에 끼치는 영향)

문서 끝 — 빠른 참고 표(요약)

- Streaming 하루(8tick): 스트레스 +9~+18 / 피로 +48 / 행복 +8~+16 / 건강 -6~+1
- Rest 하루(8tick): 스트레스 -24~-40 / 피로 -34~-54 / 행복 +10~+15 / 건강 +19~+29

- 자동 시뮬레이터(가짜 플레이어가 여러 스케줄을 반복 실행하여 수치 수집) 스크립트 초안
